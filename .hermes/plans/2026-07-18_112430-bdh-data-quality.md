# BDH Data Quality & Association Implementation Plan

> **For Hermes:** implement this plan task-by-task with strict TDD. Backend only; do not touch the visualization/UI branch.

**Goal:** remove structural self-loops, stop ingesting derived external artifacts, control the weak Hebbian tail without deleting fresh memory, and integrate neurogenesis notes through validated source-node associations instead of guessed wikilinks.

**Architecture:** keep the current separation between structural graph edges, Hebbian synapses, and neurogenesis provenance. Structural hygiene is enforced at graph-build time; external-source hygiene is enforced at the source scanner; weak-synapse cleanup belongs to consolidation, not the online query path; neurogenesis provenance is persisted as validated node IDs in frontmatter and materialized as generated graph edges only when both endpoints resolve.

**Tech Stack:** Python 3.11+, aiohttp, ChromaDB, Obsidian Markdown/YAML frontmatter, pytest, existing multi-vault/federated graph runtime.

2. External-source exclusion of derived artifacts such as `test-results/error-context.md`.
3. Weak Hebbian synapse retention/cleanup and observability.

### Explicitly out of scope

- Any UI/3D renderer/template change.
- Reintroducing guessed Markdown wikilinks into generated notes.
- Rewriting existing quarantined notes in place.
- Changing the online Hebbian learning formula before the retention policy is measured.
- Deleting live state without a backup and a dry-run report.

## Design decisions

### D1 — Self-loops are not knowledge edges

A resolved wikilink whose target resolves to the same node as its source is ignored by both graph builders. It should not become an unresolved link and should not be represented as a structural synapse. The builder logs a count for auditability.

### D2 — Derived artifacts are excluded centrally

Add derived-artifact patterns to `DEFAULT_EXTERNAL_EXCLUDES` so every `ExternalMarkdownSource` gets safe defaults. Keep per-source config excludes as an additional layer. The local runtime config also gets explicit excludes for clarity, but code defaults remain authoritative for new deployments.

Initial patterns:

```yaml
- "**/test-results/**"
- "**/coverage/**"
- "**/.next/**"
- "**/.cache/**"
- "**/playwright-report/**"
- "**/htmlcov/**"
```

Do not exclude valid documentation directories such as `docs/audits/` unless a concrete artifact pattern proves necessary.

### D3 — Influence threshold and retention threshold are different

`hebbian_min_score = 0.15` remains the online creation/ranking threshold. It must not be reused as a destructive persistence threshold. A weak synapse can be fresh and worth retaining, or old and disposable.

Consolidation gets a stale-weak policy based on all three signals:

```text
weight < weak_weight_threshold
AND frequency <= weak_max_frequency
AND last_coactivated older than weak_min_age_hours
```

Recommended initial defaults:

```yaml
consolidation_weak_weight_threshold: 0.15
consolidation_weak_max_frequency: 1.0
consolidation_weak_min_age_hours: 48
```

The policy runs only during explicit consolidation, never during a live query. Existing structural pruning below `0.02` remains as the hard floor.

### D4 — Neurogenesis provenance uses node IDs, not titles

Generated notes receive a JSON-compatible frontmatter field:

```yaml
activated_from_ids: ["vault:wiki/concepts/source.md", "external:projects/demo/README.md"]
```

The IDs are captured from the actual `active` dictionary, not reconstructed from titles. On graph build, each ID is resolved exactly. Missing IDs are ignored and counted as unresolved generated provenance; no basename fallback and no guessed wikilink are allowed.

For every valid source ID, the graph materializes generated edges in both directions:

```python
{
    "target": source_id,
    "type": "neurogenesis_source",
    "relation": "activated_from",
    "weight": 0.35,
    "explicit": False,
    "generated": True,
    "traversable": True,
}
```

The reciprocal edge lets retrieval reach a newborn note from its source context while preserving that this is provenance, not a user-authored wikilink.
